-------------------------------------------

Quiero que el repositorio permita ejecutar la app localmente en mi ordenador con RStudio.

El flujo local debe ser:

1. Clonar el repositorio.
2. Abrir el proyecto en RStudio.
3. Restaurar dependencias con renv.
4. Ejecutar la app.
5. Que Shiny levante un localhost automáticamente.
6. Que pueda interactuar con filtros, mapas, gráficos y tablas.

También quiero un workflow manual en GitHub Actions que sirva para comprobar que el proyecto está bien montado. Sé que GitHub Actions no puede abrirme visualmente un localhost en mi navegador local, porque se ejecuta en un runner remoto. Por tanto, NO intentes prometer algo imposible.

Lo que debe hacer el workflow manual es:

- instalar R;
- restaurar dependencias;
- generar datos si no existen;
- comprobar que los scripts cargan;
- comprobar que la app puede arrancar;
- ejecutar validaciones básicas;
- opcionalmente lanzar la app durante unos segundos en el runner para verificar que no crashea;
- finalizar correctamente.

Además, crea instrucciones claras en el README para ejecutar el localhost en local desde RStudio o terminal.

Requisitos funcionales de la app
--------------------------------

La app debe tener una interfaz clara en catalán.

Título principal:

"Indicadors Territorials de Salut i Demografia"

Subtítulo:

"Aplicació demostrativa en R Shiny per a l'exploració d'indicadors territorials"

Debe tener un sidebar o panel lateral con filtros:

- Any: selector de año, por ejemplo 2020, 2021, 2022, 2023, 2024.
- Comarca: selector múltiple.
- Municipi: selector múltiple dependiente de la comarca seleccionada.
- Indicador: selector, por ejemplo:
  - Població
  - Renda mitjana
  - Taxa d'envelliment
  - Incidència sanitària
  - Índex de vulnerabilitat
  - Accessibilitat a serveis
- Botón "Actualitzar".
- Botón "Reiniciar filtres".

Debe haber KPIs principales:

- Nombre de municipis seleccionats.
- Població total.
- Renda mitjana ponderada.
- Taxa d'envelliment mitjana.
- Incidència sanitària mitjana.
- Índex de vulnerabilitat mitjà.

Debe haber un mapa interactivo con leaflet:

- Polígonos territoriales ficticios o puntos si es más sencillo.
- Preferiblemente usar sf.
- Colorear por indicador seleccionado.
- Tooltip con municipio, comarca, año e indicadores.
- Leyenda.
- Zoom.
- Vista limpia.

Si generar polígonos realistas es demasiado complejo, puedes crear geometrías sf simuladas en forma de cuadrícula territorial. Lo importante es demostrar integración con sf + leaflet.

Debe haber gráficos con plotly:

- Evolución temporal del indicador seleccionado.
- Ranking de municipios.
- Comparación por comarca.
- Distribución del indicador seleccionado.

Debe haber tabla con reactable:

- Tabla filtrable, ordenable y legible.
- Mostrar municipio, comarca, año e indicadores.
- Formato numérico correcto.
- Resaltar algunas columnas si tiene sentido.

Debe haber descarga:

- Botón para descargar CSV de los datos filtrados.
- Nombre de archivo con fecha.

Debe haber una pestaña de metodología:

- Explicar que los datos son simulados.
- Explicar el objetivo de la app.
- Explicar las variables.
- Explicar limitaciones.
- Explicar cómo se estructura el código.

Datos
-----

Puedes crear datos sintéticos.

Necesito que generes un script reproducible:

scripts/01_generate_demo_data.R

Ese script debe crear:

1. Un dataset tabular de indicadores:
   - municipi_id
   - municipi
   - comarca
   - any
   - poblacio
   - renda_mitjana
   - taxa_envelliment
   - incidencia_sanitaria
   - index_vulnerabilitat
   - accessibilitat_serveis

2. Un objeto geoespacial sf:
   - geometría territorial ficticia;
   - municipi_id;
   - municipi;
   - comarca.

Puedes guardar los datos procesados en:

data/processed/indicadors_territorials.rds
data/processed/geometries_municipals.rds

Los datos deben estar suficientemente bien creados para que los filtros, mapas, tablas y gráficos tengan sentido.

Ejemplo de lógica:

- 40-80 municipios ficticios.
- 6-10 comarcas.
- 5 años.
- Variables generadas con coherencia:
  - población positiva;
  - renta media entre 18.000 y 45.000;
  - envejecimiento entre 10% y 35%;
  - incidencia sanitaria entre 50 y 500;
  - vulnerabilidad entre 0 y 100;
  - accesibilidad entre 0 y 100.
- Crear cierta correlación entre variables:
  - mayor vulnerabilidad puede relacionarse con menor renta;
  - mayor envejecimiento puede relacionarse con mayor incidencia;
  - mejor accesibilidad puede relacionarse con menor vulnerabilidad.
